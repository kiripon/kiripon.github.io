Extensible Effects と Data Types a la carte の関係

Posted on none
Tags: Haskell

http://okmij.org/ftp/Haskell/extensible/で触れられているように,Extensible Effects と Data Types a la carte の手法は

> {-# LANGUAGE DeriveFunctor ,UnicodeSyntax,GADTs,RankNTypes#-}
> module DataTypesALaCarte where
> import Control.Applicative
> import Data.Typeable

Functorの最小不動点をとったものがFix fであった。

> data Fix f = Fix f (Fix f) deriving Functor

これに加えて,値aを表すラベル　Pure a との直和をとったものについての最小不動点をとったものがFreeとよばれる構造になる。

> data Free f a = Pure a
>               | Free (f (Free f a))

この構造はMonadにできる。

> instance Functor f => Functor (Free f) where
>   fmap f x = case x of
>     Pure a -> Pure (f a)
>     Free a -> Free (fmap (fmap f) a)
>     
> instance Functor f => Applicative (Free f) where
>   pure = Pure
>   Pure f <*> x = fmap f x
>   Free f <*> x = Free (fmap (<*>x) f)
>   
> instance Functor f => Monad (Free f) where
>   return = Pure
>   Pure x >>= y = y x
>   Free x >>= y = Free (fmap (>>=y) x)

これが Free monad とよばれる構造になる。

functorの集合を考えた時、これの不動点をとったものが次のVEになる。

> data Union r v = ∀ t. (Functor t,Typeable t) => Union (t v)
> data VE r w = Val w
>             | E !(Union r (VE r w))

さらに、これをCPS(継続渡し)で書きなおしたものがEffモナドになる。

> newtype Eff r a = Eff {runEff :: ∀ w. (a -> VE r w) -> VE r w }

インスタンス宣言は以下のようになる。

> instance Functor (Eff r) where
>   fmap f m = Eff (\k -> runEff m (k . f))
> instance Applicative (Eff r) where
>   pure x  = Eff (\k -> k x)
>   f <*> x = Eff (\k -> runEff f (\f' -> runEff (fmap f' x) k))
> instance Monad (Eff r) where
>   return x = Eff (\k->k x)
>   m >>= p  = Eff (\k -> runEff m (\m' -> runEff (p m') k))

CPSでかかれているので、この宣言にはデータ構築子は出現しない。 結果を取り出すには、継続にidと同型なデータ構築子を渡してやればいい。

> admin :: Eff t w -> VE t w
> admin (Eff m) = m Val

値全体を取り出すには,adminで値を取り出して、その上でfromValで値を取り出してやればいい。

> fromVal :: VE w a -> a
> fromVal (Val x) = x

つまり以下のrun'関数をつかえばいい。

> run' :: Eff w c -> c
> run' = fromVal . admin

しかしこれだけではすべての Effect が実行されたかどうかを保証できない。 そこでEff w cの型パラメータwを()として、型クラスによる制約がなにもない時に限ってrunが 駆動できるようにする。

> run :: Eff () c -> c
> run = fromVal . admin
